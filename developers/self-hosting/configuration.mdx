Configure Twenty to match your infrastructure, security requirements, and integration needs.

## Configuration Overview

Twenty is configured primarily through environment variables. Configuration can be set in:

- `.env` file (local development)
- Docker Compose `.env` file
- Kubernetes ConfigMaps and Secrets
- System environment variables

## Core Configuration

### Server Settings

<ParamField path="NODE_ENV" type="string" default="development">
  Environment mode: `development` or `production`
</ParamField>

<ParamField path="SERVER_URL" type="string" required>
  Public URL where Twenty is accessible (e.g., `https://crm.yourcompany.com`)
</ParamField>

<ParamField path="FRONTEND_URL" type="string" default="http://localhost:3001">
  Frontend application URL (for development)
</ParamField>

<ParamField path="PORT" type="number" default="3000">
  Port for the server to listen on
</ParamField>

<ParamField path="APP_SECRET" type="string" required>
  Secret key for encryption and JWT signing. Must be at least 32 characters.
</ParamField>

<Tip>
  Generate a secure `APP_SECRET`:
  ```bash
  openssl rand -base64 32
  ```
</Tip>

## Database Configuration

### PostgreSQL

<ParamField path="PG_DATABASE_URL" type="string" required>
  PostgreSQL connection string:
  ```
  postgres://username:password@host:port/database
  ```
</ParamField>

<ParamField path="PG_SSL_ALLOW_SELF_SIGNED" type="boolean" default="false">
  Allow self-signed SSL certificates for PostgreSQL connections
</ParamField>

### Redis

<ParamField path="REDIS_URL" type="string" required>
  Redis connection string:
  ```
  redis://host:port
  ```
  Or with authentication:
  ```
  redis://:password@host:port
  ```
</ParamField>

### ClickHouse (Optional)

<ParamField path="ANALYTICS_ENABLED" type="boolean" default="false">
  Enable analytics with ClickHouse
</ParamField>

<ParamField path="CLICKHOUSE_URL" type="string">
  ClickHouse connection string for analytics:
  ```
  http://default:password@localhost:8123/twenty
  ```
</ParamField>

## Storage Configuration

### Local Storage

<ParamField path="STORAGE_TYPE" type="string" default="local">
  Storage backend: `local` or `s3`
</ParamField>

<ParamField path="STORAGE_LOCAL_PATH" type="string" default=".local-storage">
  Path for local file storage
</ParamField>

### S3 Storage

For production deployments, S3-compatible storage is recommended:

<ParamField path="STORAGE_TYPE" type="string">
  Set to `s3` to enable S3 storage
</ParamField>

<ParamField path="STORAGE_S3_REGION" type="string">
  AWS region (e.g., `us-east-1`, `eu-west-3`)
</ParamField>

<ParamField path="STORAGE_S3_NAME" type="string">
  S3 bucket name
</ParamField>

<ParamField path="STORAGE_S3_ENDPOINT" type="string">
  Custom S3 endpoint for S3-compatible services (MinIO, DigitalOcean Spaces, etc.)
</ParamField>

<CodeGroup>
```bash AWS S3
STORAGE_TYPE=s3
STORAGE_S3_REGION=us-east-1
STORAGE_S3_NAME=twenty-uploads
```

```bash MinIO
STORAGE_TYPE=s3
STORAGE_S3_REGION=us-east-1
STORAGE_S3_NAME=twenty
STORAGE_S3_ENDPOINT=https://minio.yourcompany.com
```

```bash DigitalOcean Spaces
STORAGE_TYPE=s3
STORAGE_S3_REGION=nyc3
STORAGE_S3_NAME=your-space-name
STORAGE_S3_ENDPOINT=https://nyc3.digitaloceanspaces.com
```
</CodeGroup>

## Email Configuration

### Email Settings

<ParamField path="EMAIL_DRIVER" type="string" default="logger">
  Email driver: `smtp` for production, `logger` for development/testing
</ParamField>

<ParamField path="EMAIL_FROM_ADDRESS" type="string">
  Sender email address (e.g., `contact@yourcompany.com`)
</ParamField>

<ParamField path="EMAIL_FROM_NAME" type="string">
  Sender display name (e.g., `John from YourCompany`)
</ParamField>

<ParamField path="EMAIL_SYSTEM_ADDRESS" type="string">
  System email address for automated messages
</ParamField>

### SMTP Configuration

<ParamField path="EMAIL_SMTP_HOST" type="string">
  SMTP server hostname (e.g., `smtp.gmail.com`)
</ParamField>

<ParamField path="EMAIL_SMTP_PORT" type="number">
  SMTP server port (typically `587` for TLS or `465` for SSL)
</ParamField>

<ParamField path="EMAIL_SMTP_USER" type="string">
  SMTP username
</ParamField>

<ParamField path="EMAIL_SMTP_PASSWORD" type="string">
  SMTP password or app-specific password
</ParamField>

<CodeGroup>
```bash Gmail
EMAIL_DRIVER=smtp
EMAIL_SMTP_HOST=smtp.gmail.com
EMAIL_SMTP_PORT=465
EMAIL_SMTP_USER=your-email@gmail.com
EMAIL_SMTP_PASSWORD=your-app-password
EMAIL_FROM_ADDRESS=your-email@gmail.com
```

```bash SendGrid
EMAIL_DRIVER=smtp
EMAIL_SMTP_HOST=smtp.sendgrid.net
EMAIL_SMTP_PORT=587
EMAIL_SMTP_USER=apikey
EMAIL_SMTP_PASSWORD=your-sendgrid-api-key
EMAIL_FROM_ADDRESS=noreply@yourcompany.com
```

```bash AWS SES
EMAIL_DRIVER=smtp
EMAIL_SMTP_HOST=email-smtp.us-east-1.amazonaws.com
EMAIL_SMTP_PORT=587
EMAIL_SMTP_USER=your-ses-username
EMAIL_SMTP_PASSWORD=your-ses-password
EMAIL_FROM_ADDRESS=noreply@yourcompany.com
```
</CodeGroup>

## Example Configurations

### Development Environment

```bash .env
NODE_ENV=development
PG_DATABASE_URL=postgres://postgres:postgres@localhost:5432/default
REDIS_URL=redis://localhost:6379
APP_SECRET=dev-secret-change-in-production
SIGN_IN_PREFILLED=true
FRONTEND_URL=http://localhost:3001
SERVER_URL=http://localhost:3000
STORAGE_TYPE=local
STORAGE_LOCAL_PATH=.local-storage
EMAIL_DRIVER=logger
```

### Production Environment

```bash .env
NODE_ENV=production
PG_DATABASE_URL=postgres://user:password@db.yourcompany.com:5432/twenty
REDIS_URL=redis://:password@redis.yourcompany.com:6379
APP_SECRET=your-secure-random-string-from-openssl
SERVER_URL=https://crm.yourcompany.com

# Storage
STORAGE_TYPE=s3
STORAGE_S3_REGION=us-east-1
STORAGE_S3_NAME=twenty-production-uploads

# Email
EMAIL_DRIVER=smtp
EMAIL_SMTP_HOST=smtp.sendgrid.net
EMAIL_SMTP_PORT=587
EMAIL_SMTP_USER=apikey
EMAIL_SMTP_PASSWORD=your-sendgrid-api-key
EMAIL_FROM_ADDRESS=noreply@yourcompany.com
EMAIL_FROM_NAME=YourCompany CRM

# Auth
AUTH_GOOGLE_ENABLED=true
AUTH_GOOGLE_CLIENT_ID=your-client-id
AUTH_GOOGLE_CLIENT_SECRET=your-client-secret
AUTH_GOOGLE_CALLBACK_URL=https://crm.yourcompany.com/auth/google/redirect

# Monitoring
EXCEPTION_HANDLER_DRIVER=sentry
SENTRY_DSN=https://xxx@xxx.ingest.sentry.io/xxx
SENTRY_ENVIRONMENT=production
LOG_LEVELS=error,warn

# Security
API_RATE_LIMITING_TTL=60000
API_RATE_LIMITING_LIMIT=100
```

```bash Check Server

```

```bash Check Database
psql $PG_DATABASE_URL -c "SELECT version();"
```

```bash Check Redis
redis-cli -u $REDIS_URL ping
```
</CodeGroup>
